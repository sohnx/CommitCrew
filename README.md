## вңЁ What is Grind Squad?

Grind Squad is a **single-file web app** that your whole CSE squad can use to stay accountable together. Built specifically for engineering students who are grinding LeetCode, building projects, and need something more than a shared spreadsheet.

It uses **GitHub Gist as a live shared database** вҖ” meaning everyone reads and writes to the same data in real time, with zero server costs, zero setup headache, and it runs perfectly on GitHub Pages.

---

## рҹҡҖ Features

### рҹ“Ҡ Dashboard
- **Daily streak tracker** with fire animations that level up at 7, 14, and 30 days
- **Monthly calendar heatmap** showing your coding activity
- **Taunt banner** вҖ” calls you out when a teammate has solved more problems than you today, or when your streak is about to break at midnight
- **Squad overview cards** вҖ” see everyone's progress, today's count, streaks, and a 14-day mini heatmap per person at a glance

### рҹ’» LeetCode Tracker
- Log Easy / Medium / Hard problems solved each day
- Add topic notes (e.g. *"practiced sliding window and two pointers"*)
- **7-day bar chart** showing your solve trend
- Difficulty breakdown bars
- Full history table with 30-entry log

### рҹӣ  Project Workspace
Full workspace per project with **5 tabs**:

| Tab | What it does |
|-----|-------------|

| **Blackboard** | Shared live notepad вҖ” post Notes, Ideas, Code Snippets, Links, or Blockers |
| **Resources** | Add GitHub repos, docs, tutorials, tools with clickable links |
| **Settings** | Set phase, domain, deadline, GitHub URL, and custom team roles |

**11 project domains** вҖ” Web Dev, AI/ML, Mobile, Security, Game Dev, Data Science, Systems, Cloud/DevOps, Blockchain, IoT, Other

**5 project phases** вҖ” рҹ’Ў Ideation вҶ’ рҹҸ—пёҸ Building вҶ’ рҹ§Ә Testing вҶ’ рҹҡҖ Deployed вҶ’ вң… Completed

### рҹҸҶ Leaderboard
- **Streak Kings** вҖ” who has the longest active streak

- **Today's Battle** вҖ” live daily competition board
- **рҹҳӨ Shame Board** вҖ” shows who broke their streak and how many days they've been slacking

### рҹ”’ Security
- **PIN-protected profiles** вҖ” SHA-256 hashed in the browser, stored in the Gist. No one can log in as you without your PIN
- **Shared Squad Token model** вҖ” one owner token writes to the Gist; friends join with the same token so GitHub always accepts writes

## вҡЎ Quick Start

### For the Squad Creator

**1.** Fork or create a new GitHub repo, upload `index.html`, enable GitHub Pages (`Settings вҶ’ Pages вҶ’ Deploy from branch вҶ’ main`)

**2.** Create a GitHub Personal Access Token:
- Go to [github.com/settings/tokens](https://github.com/settings/tokens)
- Click **Generate new token (classic)**
- Check only the **`gist`** scope
- Copy the token

**3.** Open your live site, choose **Create Squad**, paste your token, and create your profile with a PIN

**4.** Share with your squad:
```
рҹҢҗ Site URL  : https://your-username.github.io/grind-squad/
рҹ—ғпёҸ Gist ID   : (shown on the "All Set" screen)
рҹ”‘ Squad Token: (shown on the "All Set" screen вҖ” keep this within the squad)
```

### For Squad Members

**1.** Open the site URL your squad creator shared

**2.** Choose **Join Squad**, paste the Squad Token + Gist ID

**3.** Create your profile with a unique name, avatar emoji, and PIN

**4.** Start grinding рҹ”Ҙ

## рҹӣЎпёҸ Privacy & Security Notes

- Your GitHub token is stored **only in your browser's localStorage** вҖ” it never leaves your device except to call the GitHub API directly
- PINs are hashed with **SHA-256 + salt** client-side before being stored in the Gist вҖ” the raw PIN is never saved anywhere
- The Gist is **private** by default вҖ” it won't appear in search results
- Anyone with the Squad Token can read/write the Gist at the API level вҖ” only share it with people you trust (same as sharing a Google Doc edit link)

## рҹӨқ How to Update

When a new version of the app is available, just replace `index.html` in your repo:

1. Go to your repo on GitHub
2. Click `index.html` вҶ’ click the **pencil (edit) icon**
3. Select all вҶ’ paste the new file contents
4. Click **Commit changes**
5. GitHub Pages redeploys in ~60 seconds вң…

Your squad's data lives in the Gist, not the file вҖ” so **updates never wipe any progress**.
